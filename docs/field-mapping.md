# Seeklight to DC / DCTerms Field Mapping

This document records the mapping between fields generated by Seeklight (from JSTOR) and the established Dublin Core (DC) and DCTerms metadata fields used in Grinnell College's digital collections.

## Status

✅ **Initial mapping complete** — Field mappings have been documented and a sample mapped dataset has been created.

## Mapped Metadata Dataset

A transformed dataset is available that demonstrates the application of these field mappings:

- **File**: [`../sample-objects/existing-dc-metadata/seeklight_export_to_DCAP01.csv`](../sample-objects/existing-dc-metadata/seeklight_export_to_DCAP01.csv)
- **Source**: Seeklight export file `LAMP_AI_624375.csv` (8 sample objects)
- **Format**: Alma/DCAP01 format matching the structure of `alma_export_20260430_104136.csv`
- **Contents**: All Seeklight-generated metadata transformed into DC/DCTerms fields using the mappings documented below

This file serves as a reference implementation of the field mappings and can be used to:
- Compare Seeklight output with existing Alma metadata
- Validate the completeness and accuracy of the mappings
- Test metadata import workflows
- Identify fields that require manual review or enhancement

## How to Use This Document

1. For each sample object, compare the Seeklight-generated metadata (in [`../sample-objects/seeklight-metadata/`](../sample-objects/seeklight-metadata/)) with the existing DC metadata (in [`../sample-objects/existing-dc-metadata/`](../sample-objects/existing-dc-metadata/)).
2. Identify which Seeklight fields correspond to which DC / DCTerms fields.
3. Record the mapping in the table below, noting any transformation or notes needed.
4. Reference [`seeklight_export_to_DCAP01.csv`](../sample-objects/existing-dc-metadata/seeklight_export_to_DCAP01.csv) for examples of applied mappings.

## Field Mapping Table

Based on comparison of Alma export (`alma_export_20260430_104136.csv`) and Seeklight export (`LAMP_AI_624375.csv`):

### Core Descriptive Fields

| Seeklight Field | DC / DCTerms Field      | Notes / Transformation Required                  |
|-----------------|-------------------------|--------------------------------------------------|
| Title[3071073] | dc:title | Direct mapping |
| Creator[3071074] | dc:creator | Direct mapping; may include multiple creators separated by pipe (&#124;) |
| Description[3071079] | dc:description | Primary description field; very detailed in Seeklight output |
| Description[3071079] | dcterms:abstract | Seeklight descriptions often function as abstracts |
| Subject[3071080] | dc:subject | Direct mapping; Seeklight uses pipe (&#124;) separators |
| Keywords[3071094] | dc:subject | Additional subject terms; can be combined with Subject field |
| Named Entities[3071093] | dc:contributor | Could map to contributor when entities are people/organizations |
| Publisher[3071084] | dcterms:publisher | Direct mapping |
| Language[3071078] | dc:language | Direct mapping; uses language names (e.g., "English") |

### Date Fields

| Seeklight Field | DC / DCTerms Field      | Notes / Transformation Required                  |
|-----------------|-------------------------|--------------------------------------------------|

| Date[3071077] | dcterms:created | When date represents creation date |
| Date[3071077] | dcterms:issued | When date represents publication date |
| Period[3071091] | dcterms:temporal | Time period (e.g., "Early 20th century", "World War II") |

## Key Observations

1. **Seeklight provides rich contextual metadata**: Style, Culture, Period fields add interpretive context not typically in DC
2. **Descriptions are extensive**: Seeklight's Description field is very detailed and analytical
3. **Coverage field structure**: Seeklight uses a pipe-delimited structure combining style, culture, period, and location
4. **Format field complexity**: Seeklight combines medium, technique, material, and measurements in structured format
5. **Named entities extraction**: Seeklight identifies people, places, organizations - can supplement subject/contributor fields
6. **Date flexibility**: Seeklight handles various date formats and approximate dates well
7. **Missing from Seeklight**: Rights statements, persistent identifiers (handles), collection relationships, provenance
