# Sample Digital Objects

This directory contains sample digital objects from Grinnell College's digital collections, along with their associated metadata.

## Directory Structure

```
sample-objects/
├── _README.md                # This file
├── existing-dc-metadata/     # Existing Dublin Core (DC) metadata for each sample object
│   └── _README.md            # DC metadata documentation
├── seeklight-metadata/       # Metadata generated by Seeklight for the same objects
│   └── _README.md            # Seeklight metadata documentation
└── objects/                  # Digital object files organized by object
```

## Adding a Sample Object

When adding a sample object:

1. Create a directory in `objects/` named with the object's Title (e.g., `Lucille Ley to Jimmy Ley - June 21, 1943/`).
2. Place the digital object file(s) in that directory.
3. Add the existing DC metadata for the object in `existing-dc-metadata/` as a file named to match the object's Title (e.g., `Lucille Ley to Jimmy Ley - June 21, 1943.md`).
4. After running the object through Seeklight, place the generated metadata in `seeklight-metadata/` using the same Title-based filename (e.g., `Lucille Ley to Jimmy Ley - June 21, 1943.json`).

## Metadata Format

Metadata files can be provided in various formats:

- **Existing DC metadata**: Currently using Markdown (`.md`) files with human-readable metadata fields. Other formats like Dublin Core XML (`<dc:...>` elements) or JSON representations are also acceptable.
- **Seeklight metadata**: Use the JSON format returned by the Seeklight API or export.

## Comparison

Once both sets of metadata are in place for a given object, use the field mapping documented in [`../docs/field-mapping.md`](../docs/field-mapping.md) to compare fields and refine the mapping between Seeklight output and our established DC/DCTerms vocabulary.
