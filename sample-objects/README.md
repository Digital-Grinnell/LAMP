# Sample Digital Objects

This directory contains sample digital objects from Grinnell College's digital collections, along with their associated metadata.

## Directory Structure

```
sample-objects/
├── README.md                 # This file
├── existing-dc-metadata/     # Existing Dublin Core (DC) metadata for each sample object
└── seeklight-metadata/       # Metadata generated by Seeklight for the same objects
```

## Adding a Sample Object

When adding a sample object:

1. Place the digital object file (or a reference to it) in this directory, using a consistent naming convention (e.g., `object-001.pdf`, `object-001.jpg`).
2. Add the existing DC metadata for the object in `existing-dc-metadata/` as a file named to match the object (e.g., `object-001.xml` or `object-001.json`).
3. After running the object through Seeklight, place the generated metadata in `seeklight-metadata/` using the same base filename (e.g., `object-001.json`).

## Metadata Format

Metadata files should be provided in XML or JSON format where possible:

- **Existing DC metadata**: Prefer Dublin Core XML (`<dc:...>` elements) or a JSON representation of DC fields.
- **Seeklight metadata**: Use the format returned by the Seeklight API or export.

## Comparison

Once both sets of metadata are in place for a given object, use the field mapping documented in [`../docs/field-mapping.md`](../docs/field-mapping.md) to compare fields and refine the mapping between Seeklight output and our established DC/DCTerms vocabulary.
